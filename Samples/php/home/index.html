PayJS .NET Samples

Overview

These samples show several different ways of using PayJS:

  • Modal — display the UI over the page content when a button or <a> is clicked
  • Inline — display the UI inside a <div> or other similar element
  • RequireJS — load the PayJS library on a page that uses RequireJS
  • Dynamic — use a simple ajax helper to facilitate lazy initialization
  • JS-Only — use the PayJS library to power your own payment forms
  • Adaptive — see the UI adapt to various container sizes
  • Tokens — tokenize payment data for later use

These samples are available on GitHub.

Testing

By default, these samples use fake merchant credentials that only accept certain credit card numbers.

The following credit card numbers can be safely used:

  • 4111-1111-1111-1111 — Visa
  • 5454-5454-5454-5454 — MasterCard
  • 6011-0009-9302-6909 — Discover
  • 3714-496353-92376 — American Express

Additionally, the following dollar amounts will produce specific results:

  • $1.xx — Approved
  • $2.xx — Declined
  • $16.xx — Error
